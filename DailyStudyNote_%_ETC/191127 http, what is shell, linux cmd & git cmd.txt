3. linux cmd

예약어란? 이미 의미가 등록되있는말, 식별자로 예약어를 쓸순없다. 식별자란 쉽게말하자면 우리가 붙여주는 이름,이는 키워드와 겹칠수없다

터미널 : shell과같은 os x 에서 쓰는 애플리케이션? 

○ pwd : 현재 dr 위치

○ ls : 현 dr 내용
   ls -alF : 현 dr 내용 + 스캔 + 상세정보
   ls -l : 자세히
     -al : 숨김파일도
     -F : dir은 끝을 / 표시 (굳이 안해도 dir은 /로 표시됨. 그럼 왜?)

○ mkdir 이름 : make directory, 현재 dr에 dr만들기
      -p 이름1/이름2.. : 마트료시카처럼 여러개만들기가능(병렬아님)

○ cd dr이름 : dr로 이동(cd만 치면 홈으로이동)
           . : 현 dr
          .. : 상위 (cd ../../ : 두단계상위dr로감)
        cd / : 루트dr로감(루트dr? 루트는 / 이거인걸안다)

○ nano hello.txt : 쉘 안의 텍스트편집기(ctrl+x로 나옴)

○ cat hello,txt : 편집기 내용보기

○ man 명령어 : 명령어의 도움말을본다(bash에서 찾을수없는 명령어?? 유닉스명령어라면 리눅스의쉘인 배쉬에서 안될수도잇나?)

○ sudo 명령어 ~ : 관리자권한으로 명령어실행(관리자권한이란?)(man과 마찬가지로 찾을수없음)
         sudo -i : 일시적으로 관리자가됨
            exit : 일반유저로복귀

○ echo <내용> : 내용을 화면에 출력
            > file.txt : 파일에 출력(file 이아니여두됨)
            >> file.txt : 파일에 추가

○ cat <파일> : 파일의 내용을 표시(긴파일을 끊어서 출력해줌>more어케씀?,글고 한글은 표시안)

○ rm <파일> : 파일삭제
 rm -rf <dr> : 좀위험한 묻지마dr삭제
    -ir <dr> : 물어보고 dr삭제

○ mv <A> <B> : 동일한파일,dr이면 이름변경
                파일이나 dr 의 위치도 

○ cp <a> <b> : a를 b란이름으로 복사,당연히 위치지정가능(path잘안됨)

3-1. git cmd

소스저장소git > 게임으로 생각하자면 간편한 중간저장. 협업하는이와 공유,수정이 용이하다

○ git status : 상태보기

○ git init : 현 dr을 git으로초기화(조심할것 dr단위로 되기에 사적인정보 조심)

○ git config --global user.name '..' : 내이름설정
                          .email '..' : 내이메일설정
                               --list : 설정한것확인
○ git add <파일이름> : stage하기, 임시저장소에 저장하는것, commit 하기전의 단계 

○ git commit -m <커밋메시지> : commit 하는것, stage 한것을 로컬저장소(최신본을 HEAD라고함)에 집어넣음 push의 전단계

○ git remote add origin <원격저장소주소,GitHub의 저장소주소> : 원격저장소 주소 등록
   git remote -v : 주소확인

○ git push origin <branch이름> : 원격저장소에 올림
